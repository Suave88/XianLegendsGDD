Battle Grid System

Tactical Positioning Combat

3×4 Grid Layout

[A1] [A2] [A3] [A4]  ← Back Row
[B1] [B2] [B3] [B4]  ← Mid Row  
[C1] [C2] [C3] [C4]  ← Front Row

Allies start left (A1-C1)
Enemies start right (A4-C4)

Key Mechanic: Position matters. Front row = more damage taken but easier to hit enemies. Back row = safer but limited range.

Character Roles

Mion (Support): Back row, buffs allies, debuffs enemies, zone control

Setsuna (Tank): Front row, protects allies, area attacks, high HP

Ayane (Mage DPS): Mid/Back row, magic damage, Guardian summons

Core Mechanics

Movement: 1-2 cells per turn. Tactical repositioning crucial.

Range: Melee (adjacent), Short (1-2 cells), Long (any distance)

Line of Sight: Some skills blocked by obstacles or allies

Combo System: Adjacent allies trigger bonus effects

Node Effects: Special cells give buffs (heal, damage, regen)

Battle Flow

Turn Order: Based on Speed stat. Visible at screen top.

Actions Per Turn: Move + Action (Attack/Skill/Item/Guard)

Battle Length: 3-7 turns average. Quick, tactical, decisive.

Victory: Defeat all enemies or complete objective

⚔️ Design Goal

Tactical without overwhelming.
Disgaea-style depth with approachable learning curve.
Position matters, but mistakes aren't punished harshly.
